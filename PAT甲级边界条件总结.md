# PAT甲级边界条件总结

- A1085

  int 最值低于测试最值，改用 long long；

  有乘积时的值可能溢出；

- A1010

  有些数值没有明说边界，往往需要按照 int 的极值定义；

  int 极值获取方法可以如下：

  ```c++
  int inf = (int(1) << 31)-1;
  ```

  比如该题中radix范围并未明说导致我默认为其范围为```2~Z``` ,结果一半样例过不了检测。

  总结：**所有未明说范围的数值都需要考虑到 ```INT_MAX``` ，但是一般不会超过 ```long long``` 的极值**；
